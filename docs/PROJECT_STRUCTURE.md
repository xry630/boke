# 项目分类管理说明

本文档说明 Boke 仓库的项目分类管理结构和规范。

## 分类原则

项目按功能和用途进行分类，便于管理和扩展：

### 📁 目录分类

```
boke/
├── backend/          # 后端服务项目
├── frontend/         # 前端应用项目  
├── mobile/           # 移动端应用项目 (未来)
├── tools/            # 开发工具和脚本 (未来)
├── shared/           # 共享库和组件 (未来)
├── docs/             # 项目文档
├── configs/          # 配置文件
├── deployment/       # 部署相关文件
└── tests/            # 测试文件 (未来)
```

## 各分类说明

### 🖥️ backend/ - 后端项目

存放所有后端服务项目，每个项目应包含：
- 独立的源代码目录
- 项目配置文件
- README.md 说明文档
- API 文档

**示例结构:**
```
backend/
├── video-master/          # 视频管理后端
│   ├── src/
│   ├── README.md
│   └── ...
└── user-service/          # 用户服务 (未来)
    ├── src/
    ├── README.md
    └── ...
```

### 🌐 frontend/ - 前端项目

存放所有前端应用项目，每个项目应包含：
- 独立的源代码目录
- package.json 依赖管理
- 构建配置文件
- README.md 说明文档

**示例结构:**
```
frontend/
├── video-admin/           # 视频管理前端
│   ├── src/
│   ├── package.json
│   ├── README.md
│   └── ...
└── mobile-app/            # 移动端前端 (未来)
    ├── src/
    ├── package.json
    ├── README.md
    └── ...
```

### 📱 mobile/ - 移动端项目 (预留)

未来用于存放移动端应用项目：
- iOS 应用
- Android 应用
- React Native/Flutter 应用

### 🛠️ tools/ - 开发工具 (预留)

未来用于存放开发工具和脚本：
- 代码生成工具
- 数据迁移脚本
- 自动化脚本

### 📦 shared/ - 共享库 (预留)

未来用于存放共享组件和库：
- 通用工具库
- UI 组件库
- 类型定义

### 📚 docs/ - 文档

存放所有项目文档：
- README.md - 项目总体说明
- 各模块详细文档
- API 文档
- 使用指南
- 部署教程

### ⚙️ configs/ - 配置文件

存放项目配置文件：
- 环境配置
- 构建配置
- 项目总结文件

### 🚀 deployment/ - 部署相关

存放部署相关文件：
- Docker 配置
- Nginx 配置
- 部署脚本
- CI/CD 配置

### 🧪 tests/ - 测试文件 (预留)

未来用于存放测试文件：
- 单元测试
- 集成测试
- E2E 测试

## 命名规范

### 目录命名
- 使用小写字母和连字符：`video-master`, `user-service`
- 避免使用下划线或驼峰命名

### 文件命名
- 文档文件：使用描述性名称，如 `PROJECT_STRUCTURE.md`
- 配置文件：遵循各技术栈的命名约定

## 新项目添加流程

1. **确定分类**：根据项目类型选择合适的目录
2. **创建目录**：在相应分类下创建项目目录
3. **添加代码**：将项目代码放入目录
4. **编写文档**：添加 README.md 说明文档
5. **更新索引**：更新根目录 README.md 的项目列表

## 文档维护

- 每个项目应包含独立的 README.md
- 公共文档放在 docs/ 目录
- 保持文档与代码同步更新

## 版本管理

- 每个项目可以独立管理版本
- 重大变更需要更新根目录的版本信息
- 使用语义化版本控制

---

这种分类管理方式有助于：
- 🔍 快速定位项目
- 🚀 便于团队协作
- 📈 支持项目扩展
- 🛡️ 降低维护成本
